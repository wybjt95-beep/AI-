# BYOK 上线说明

BYOK 是 Bring Your Own Key，意思是每个用户使用自己的模型 API Key。

## 当前已完成的骨架

- 新增账号接口：注册、登录、退出、读取当前用户。
- API 配置改为用户级配置：每个登录用户保存自己的 Base URL、API Key、模型名。
- 模型调用不再读取全局 `.env` 里的用户 API Key。
- 未登录或未配置 API 时，只走本地 mock，不会消耗任何真实 Key。
- 项目看板支持按用户保存项目；当前本地版保存到 SQLite 数据库 `网页平台/.data/app.db`。
- 后端接口已做登录保护，未登录不能直接调用剧本分析或拆分镜接口。

## 上线时必须替换的部分

- 把本地 SQLite `网页平台/.data/app.db` 换成正式托管数据库，例如 PostgreSQL / MySQL / Supabase。
- 把本地加密函数换成云厂商 KMS 或成熟密钥管理方案。
- 开启 HTTPS，并给 Cookie 加 `Secure`。
- 增加接口限流、日志脱敏、错误监控。
- 不要上传本地 `.env`、`.data/` 或任何真实用户 Key。

## 用户使用方式

1. 用户注册或登录。
2. 打开 API 接入。
3. 填写自己的 API Base URL、API Key、文本模型名。
4. 保存后，该用户的剧本分析和拆分镜会使用自己的 Key。

## 部署边界

当前仍是本地 MVP 骨架，适合演示 BYOK 架构和业务流程；要正式多人上线，还需要数据库、对象存储、生产鉴权、HTTPS 和运维监控。
